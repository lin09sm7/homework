一、檔案結構概述

本系統資料庫採用 SQLite，啟用 WAL（Write-Ahead Logging）模式支援並行存取。資料庫檔案位於 database/hospital.db。所有資料表均啟用外鍵約束（PRAGMA foreign_keys = ON）。

參考文件：慈濟醫院門診掛號系統檔案結構。本系統從慈濟醫院掛號系統之檔案結構中擷取核心概念，簡化設計為六張資料表。

本系統所有資料表均採用自然鍵（Natural Key）作為主鍵，不使用自動遞增流水號，使資料具備業務意義且易於辨識。

與慈濟系統對應關係

本系統資料表慈濟系統對應檔案說明
cities縣市主檔（COUNTYTBL）儲存全台灣縣市資料
districts鄉鎮主檔（TOWNTBL）儲存各縣市下的區域資料
departments院內科別檔（GenSectionTbl）儲存醫院門診科別
doctors醫師檔（GenDoctorTbl）儲存各科別醫師
patients（新設計）整合病患基本資料
registrations掛號看診檔（OpdRegDiagnosisTbl）儲存掛號紀錄

資料表關聯圖

  cities ──< districts           （一個縣市有多個區域）
  departments ──< doctors        （一個科別有多位醫師）
  cities ──< patients >── districts  （病患地址關聯縣市與區域）
  patients ──< registrations >── departments
                             └──── doctors
  （──< 表示一對多關聯）

台灣身分證字號驗證規則

1. 字母對照表：第一碼英文字母對應數字

ABCDEFGHIJKLM
10111213141516173418192021
NOPQRSTUVWXYZ
22352324252627282932303133

2. 檢查碼演算法（MOD-10）：

步驟說明範例（A123456789）
1字母轉數字，拆為十位數(n1)與個位數(n2)A=10 → n1=1, n2=0
2sum = n1 + n2 × 9sum = 1 + 0×9 = 1
3第 2~9 碼乘以加權值 [8,7,6,5,4,3,2,1]1×8 + 2×7 + 3×6 + 4×5 + 5×4 + 6×3 + 7×2 + 8×1 = 120
4sum += 各乘積之和sum = 1 + 120 = 121
5sum += 第 10 碼（檢查碼）sum = 121 + 9 = 130
6sum % 10 === 0 → 合法130 % 10 = 0 ✓ 合法

十、UX/UI 設計總覽

本系統採用單頁式應用（SPA）設計，所有掛號操作在同一畫面完成，無需切換頁面，提升掛號人員作業效率。整體風格以醫療機構的專業、清晰、易讀為設計原則。

十七、系統功能流程

掛號作業流程

步驟 1：輸入病歷號碼
掛號人員輸入病患的台灣身分證字號（10碼），系統即時驗證格式與檢查碼。
↓
步驟 2：系統自動查詢
驗證通過後，系統自動呼叫 GET /api/patients?medical_no=xxx 查詢病患資料。
↓
步驟 3A：既有病患 → 自動帶入
查到既有病患：自動帶入姓名、性別、生日、年齡、電話、縣市、區域、地址，初複診設為「複診」。
步驟 3B：新病患 → 手動填寫
查無此病患：初複診設為「初診」，掛號人員手動填寫病患資料。
↓
步驟 4：選擇門診資訊
選擇掛號日期（預設當天）→ 選擇掛號時段（上午／下午）→ 選擇科別 → 系統自動載入醫師名單 → 選擇醫師。
↓
步驟 5：確認掛號
點擊「確認掛號」→ 系統驗證必填欄位 → 新增/更新病患資料（POST/PUT /api/patients）→ 建立掛號紀錄（POST /api/registrations）→ 顯示掛號號碼。
↓
步驟 6：列印或清除
可點擊「列印掛號單」列印收據，或點擊「清除畫面」準備下一位病患掛號。
