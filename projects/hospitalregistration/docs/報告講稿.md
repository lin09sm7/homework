# 安康綜合醫院掛號系統 — 報告講稿

---

## 一、開場（約 1 分鐘）

老師好，同學好，我今天要報告的專案是**「安康綜合醫院掛號系統」**。

這是一個模擬醫院門診掛號流程的全端網頁應用程式。使用者可以透過這個系統輸入病患資料、選擇看診科別與醫師，完成掛號後會自動產生掛號號碼。

整個系統的設計理念是：**貼近真實醫院掛號櫃台的操作流程**，讓掛號人員可以在同一個畫面上完成所有操作。

---

## 二、技術架構（約 2 分鐘）

這個專案採用**前後端分離**的架構，總共分成三個部分：

### 前端（Frontend）

- 使用 **React 19** 搭配 **TypeScript** 開發
- 樣式框架使用 **Tailwind CSS v4**
- 打包工具使用 **Vite 8**
- 圖示使用 **Lucide React** 圖示庫
- API 請求使用 **Axios**

選擇 React 是因為它的元件化開發方式很適合這種表單系統，每個輸入欄位都可以獨立成元件來複用。TypeScript 則是幫助我在開發時就能檢查型別錯誤，減少 bug。

### 後端（Backend）

- 使用 **Node.js** 搭配 **Express 5** 框架
- 提供 **RESTful API** 給前端呼叫
- 運行在 **port 3001**

### 資料庫（Database）

- 使用 **SQLite**，透過 **better-sqlite3** 套件操作
- 選擇 SQLite 的原因是它不需要另外安裝資料庫伺服器，一個 `.db` 檔案就是整個資料庫，方便部署和展示

---

## 三、資料庫設計（約 2 分鐘）

資料庫共有 **5 張資料表**，我用 ER 的概念來說明它們之間的關聯：

### 1. cities（縣市表）

- 存放全台灣 22 個縣市
- 主鍵是縣市名稱（name）

### 2. districts（區域表）

- 存放各縣市的鄉鎮市區
- 複合主鍵：city_name + name
- 透過**外鍵**關聯到 cities 表

### 3. departments（科別表）

- 存放 8 個科別：內科、外科、小兒科、婦產科、耳鼻喉科、皮膚科、眼科、家醫科
- 主鍵是科別代碼（code），例如 `internal`、`surgery`

### 4. doctors（醫師表）

- 每個科別有 3 位醫師，共 24 位
- 複合主鍵：department_code + name
- 透過外鍵關聯到 departments 表

### 5. patients（病患資料表）

- 病歷號碼（medical_no）作為主鍵，實際上就是身分證字號
- 包含姓名、性別、生日、年齡、電話、地址等欄位
- 透過外鍵關聯到 cities 和 districts

### 6. registrations（掛號紀錄表）

- 這是最核心的表，記錄每一筆掛號
- 複合主鍵：**掛號日期 + 時段 + 科別 + 序號**
- 透過外鍵關聯 patients、departments、doctors 三張表
- 掛號狀態有三種：confirmed（已確認）、cancelled（已取消）、completed（已完成）

這些表之間的關聯確保了**資料的一致性**，例如：掛號時選的醫師一定要屬於選的科別，不會出現選了內科卻指定外科醫師的情況。

## 四、API 設計（約 1.5 分鐘）

後端總共提供 **7 支 API**，全部都是 RESTful 風格：

| 方法 | 路徑 | 功能 |
|------|------|------|
| GET | `/api/cities` | 取得所有縣市 |
| GET | `/api/cities/:cityName/districts` | 取得某縣市的所有區域 |
| GET | `/api/departments` | 取得所有科別 |
| GET | `/api/departments/:deptCode/doctors` | 取得某科別的所有醫師 |
| GET | `/api/patients?medical_no=xxx` | 用病歷號碼查詢病患 |
| POST | `/api/patients` | 新增病患資料 |
| PUT | `/api/patients/:medicalNo` | 更新病患資料 |
| POST | `/api/registrations` | 建立掛號紀錄 |
| GET | `/api/registrations` | 取得所有掛號紀錄 |

API 的設計有幾個重點：

- **URL 結構有層次**：例如 `/api/cities/台北市/districts` 表示取得台北市底下的區域，很直覺
- **使用參數化查詢**（Prepared Statement）防止 SQL Injection 攻擊
- **錯誤處理**：例如新增重複病歷號碼會回傳 409 Conflict

## 五、前端功能展示（約 3 分鐘）

接下來我實際操作展示這個系統的功能。

### 功能 1：身分證字號驗證

病歷號碼欄位其實就是身分證字號。系統有實作完整的**台灣身分證字號驗證**：

- 第一碼必須是英文字母
- 第二碼起必須是數字
- 總共 10 碼
- 最後會做 **checksum 檢查碼驗證**，用加權運算確認是合法的身分證號

如果輸入的格式不對，會即時顯示紅色的錯誤訊息，例如「還需輸入 3 碼」或「檢查碼錯誤」。

### 功能 2：自動帶入病患資料 & 初複診判斷

- 當輸入一個**已存在**的病歷號碼，系統會自動從資料庫帶出病患的姓名、性別、生日等所有資料，並自動設定為**「複診」**
- 如果是**新的**病歷號碼，則自動設定為**「初診」**，讓掛號人員手動填寫資料

### 功能 3：級聯下拉選單（Cascade Dropdown）

系統有兩組級聯選單：

1. **縣市 → 區域**：選了「台北市」，區域下拉選單才會載入台北市的 12 個區
2. **科別 → 醫師**：選了「內科」，醫師下拉選單才會顯示內科的 3 位醫師

這是透過 React 的 `useEffect` 監聽上層選單的變化，再呼叫 API 動態載入下層選項。

### 功能 4：生日自動計算年齡

選擇生日之後，年齡欄位會**自動計算**，不需要手動輸入。計算邏輯有考慮到「今年還沒過生日」的情況，會少算一歲。

### 功能 5：掛號號碼自動產生

點選「確認掛號」後，系統會根據**當天日期 + 時段 + 科別**去計算目前已掛了幾號，自動產生下一個號碼。例如今天上午內科已經掛了 3 個人，下一位就是 04 號。

### 功能 6：四個操作按鈕

- **確認掛號**：驗證必填欄位 → 新增或更新病患資料 → 建立掛號紀錄 → 顯示號碼
- **列印掛號單**：呼叫瀏覽器的列印功能
- **清除畫面**：重設所有欄位回初始狀態
- **結束離開**：關閉視窗

## 六、UI 設計重點（約 1 分鐘）

- 採用**大地色系**（綠色 #7C9070 為主色），營造醫院系統沉穩、專業的視覺感受
- 使用 **Noto Sans TC** 中文字型，確保中文顯示效果良好
- 畫面分為三個區塊：**病患資訊**、**門診預約**、**操作按鈕**，資訊層次分明
- 掛號號碼用大字體突出顯示，方便掛號人員一眼看到
- 所有表單元件（輸入框、下拉選單、日期選擇器）都是自己封裝的共用元件，確保風格一致

## 七、啟動方式（約 30 秒）

為了方便使用，我做了兩個 `.command` 腳本檔：

1. **install.command**：雙擊即可自動安裝前端和後端的所有套件
2. **start.command**：雙擊即可同時啟動前端（port 5173）和後端（port 3001），並自動開啟瀏覽器

按 `Ctrl + C` 就可以同時停止前後端伺服器。這樣即使不熟悉指令的人也可以輕鬆執行。

總結一下這個專案的特色：

1. **前後端分離架構**：React + Express + SQLite，三層職責明確

3. **智慧化操作**：自動帶入舊病患資料、自動判斷初複診、自動計算年齡、自動產生掛號號碼

5. **友善的使用體驗**：級聯下拉選單、即時錯誤提示、一鍵啟動
